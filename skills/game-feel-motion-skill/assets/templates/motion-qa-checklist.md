# Motion QA Checklist

## Verdict
- [ ] Approve
- [ ] Block

## Motion Spec
- [ ] Event is defined.
- [ ] Feel goal is defined.
- [ ] Motion vocabulary name is specific.
- [ ] Duration/easing/spring/frame count is explicit.
- [ ] Reduced-motion fallback is defined when needed.

## Asset Spec
- [ ] Required source assets are listed.
- [ ] Game-ready exports are listed.
- [ ] Sequential assets have fixed frame size.
- [ ] Pivot is defined.
- [ ] Baseline is defined when relevant.
- [ ] Gap and margin are defined.
- [ ] Internal padding is defined.

## Spacing
- [ ] Sheet width matches formula.
- [ ] Sheet height matches formula.
- [ ] Frames do not overlap.
- [ ] Glow/trails/shadows stay inside cells.
- [ ] No text, labels, watermark, or final grid lines.

## In-Game
- [ ] Readable at gameplay distance.
- [ ] Does not hide important state.
- [ ] Feels responsive under repeated input.
- [ ] Works on real target background.
- [ ] Performance is acceptable.
