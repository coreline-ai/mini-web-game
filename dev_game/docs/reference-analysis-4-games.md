# Reference Analysis — 4 Mobile/Arcade Games

> 목적: `Don't Get Pooped!` 같은 모바일/웹 아케이드 게임을 반복 제작하기 위한 템플릿을 만들기 전에, 성공한 레퍼런스의 공통 제작 원리를 추출한다.

## 0. 병렬 분석 할당 결과

| 에이전트 | 담당 게임 | 분석 초점 | 주요 결론 |
|---|---|---|---|
| Agent A | Crossy Road | 8-bit endless arcade, 수집 메타 | 즉시 이해되는 조작 + 공정한 위험 + 캐릭터 수집 |
| Agent B | Jetpack Joyride | 원터치 endless runner, 미션/파워업 | 30~90초 세션 + 미션/코인/파워업으로 재도전 유도 |
| Agent C | Survivor.io | 한 손 생존 로그라이트, 성장/스킬 | 이동만으로 판단하고, 세션 내 성장 선택이 몰입을 만든다 |
| Agent D | Super Hexagon | 미니멀 고난이도 패턴/리듬 | 조작은 극단적으로 단순, 재미는 패턴·리듬·즉시 재시도에서 발생 |

## 1. 4개 게임 공통 핵심

| 공통 패턴 | 설명 | 템플릿 반영 |
|---|---|---|
| 한 문장으로 설명되는 조작 | 탭, 홀드, 좌우, 이동 하나로 시작 가능 | 새 게임 시작 시 `입력 1개 원칙` 먼저 검증 |
| 30초 안에 루프 완성 | 시작 → 위험 → 피드백 → 보상 → 실패/재시도 | `30초 루프 표`를 GDD 필수 항목으로 고정 |
| 실패 비용 최소화 | 사망 후 바로 재시작 가능 | 결과창은 짧게, Retry는 1탭 |
| 읽을 수 있는 위험 | 고난이도라도 사전 예고/패턴성이 있음 | 랜덤 즉사 금지, 예고 마커/경고음 도입 |
| 점수 + 메타 보상 분리 | 점수는 실력, 코인/스킨은 장기 동기 | 코어 밸런스를 해치지 않는 코스메틱 우선 |
| 에셋 판독성 | 위험/보상/플레이어가 0.2초 안에 구분됨 | 색상·실루엣·크기 규칙을 스타일가이드화 |
| 오디오가 행동을 강화 | 코인, 경고, 피격, 기록 갱신음이 즉시 구분 | Audio Manifest에 trigger/priority/volume 포함 |
| 모바일 세션 설계 | 짧은 플레이, 한 손, 화면 가림 최소화 | 평균 30~90초 MVP, 1~3분 확장 목표 |

## 2. Crossy Road 분석

### 2.1 핵심 한줄 정의

| 항목 | 내용 |
|---|---|
| 정의 | 8-bit/복셀풍 세로형 무한 회피·횡단 아케이드 |
| 성공 공식 | 즉시 이해되는 조작, 짧은 사망/재시작, 점진적 압박, 캐릭터 수집 |
| 제작 관점 | 규칙은 단순하지만 위험 패턴이 다양해서 매판 다르게 느껴진다 |

### 2.2 30초 코어 루프

| 시간 | 플레이어 경험 | 설계 의미 |
|---:|---|---|
| 0~3초 | 탭/스와이프로 이동 시작 | 튜토리얼 없이 규칙 이해 |
| 3~10초 | 차, 강, 기차 등 위험 패턴 관찰 | 위험을 읽고 타이밍 판단 |
| 10~20초 | 안전한 경로로 전진, 코인/점수 획득 | 작은 성취와 기록 갱신 욕구 |
| 20~30초 | 더 빠르고 복잡한 패턴 등장 | 긴장 상승 |
| 실패 후 | 즉시 결과 확인 후 재시작 | “한 판 더” 구조 |

### 2.3 시스템 분석

| 항목 | Crossy Road 방식 | 제작 템플릿 포인트 |
|---|---|---|
| 조작 | 탭 전진, 스와이프 4방향 | 한 손 조작이지만 선택지는 명확함 |
| 카메라 | 진행 방향 앞을 많이 보여주는 세로 진행 | 위험을 미리 읽을 시야 확보 |
| 난이도 | 속도, 패턴, 공간 압박, 정지 압박 | 단순 속도 증가만 쓰지 않기 |
| 실패 | 차, 물, 추격 등 원인이 명확 | 사망 원인이 명확해야 재도전 가능 |
| 메타 | 캐릭터·월드 수집 | 성능 강화보다 코스메틱 수집이 안전 |
| 아트 | 낮은 디테일, 강한 실루엣, 밝은 색 | 모바일 판독성 우선 |

### 2.4 Don't Get Pooped! 적용

| 차용 요소 | 적용안 |
|---|---|
| 캐릭터/배경 수집 | 현재 상점 구조와 잘 맞음 |
| 위험 패턴 다양화 | 똥 낙하를 완전 랜덤이 아니라 패턴 블록화 |
| 실패 원인 명확화 | 피격 시 어떤 똥에 맞았는지 색/이펙트로 표시 |
| 공정성 | 플레이어 근처 안전 통로, 예고 그림자, 작은 히트박스 유지 |

## 3. Jetpack Joyride 분석

### 3.1 핵심 한줄 정의

| 항목 | 내용 |
|---|---|
| 정의 | 원터치 상승/하강으로 장애물을 피하고 코인·토큰·미션을 수집하는 횡스크롤 endless runner |
| 성공 공식 | 원터치 조작, 즉시 피드백, 수집/미션/상점 메타, 변신/차량으로 반복 피로 감소 |
| 제작 관점 | 조작은 하나지만 세션 중 보상 밀도가 높다 |

### 3.2 30초 코어 루프

| 단계 | 행동 | 피드백 | 보상 |
|---|---|---|---|
| 시작 | 터치로 상승, 손 떼면 하강 | 즉시 캐릭터 반응 | 1초 안에 조작 이해 |
| 회피 | 레이저/미사일/전기 장애물 회피 | 경고, 폭발, 근접 긴장 | 거리 기록 |
| 수집 | 코인/토큰/파워업 획득 | 수집음, 변신, 보호 효과 | 상점/업그레이드 |
| 변주 | 차량/파워업으로 조작감 변화 | 다른 리듬 | 반복 피로 감소 |
| 사망 | 충돌 후 결과 | 코인·미션 정산 | 재도전 |

### 3.3 시스템 분석

| 항목 | Jetpack Joyride 방식 | 제작 템플릿 포인트 |
|---|---|---|
| 조작 | 원터치 홀드 | 한 손가락 하나의 의사결정 |
| 세션 | 초반 수십 초, 숙련 시 수분 | MVP 평균 30~90초 목표가 적합 |
| 난이도 | 속도/장애물 밀도/경고 패턴 증가 | 위험은 경고 후 진입해야 공정함 |
| 보상 | 코인, 스핀 토큰, 미션, 파워업 | 죽어도 남는 보상이 있어야 함 |
| 메타 | 의상, 장비, 차량, 미션 랭크 | MVP는 미션 3개 + 스킨 정도가 적절 |
| 오디오 | 코인/폭발/제트팩/슬롯 피드백 | 행위별 SFX 분리가 중요 |

### 3.4 Don't Get Pooped! 적용

| 차용 요소 | 적용안 |
|---|---|
| 미션 3개 동시 제공 | “30초 생존”, “Near Miss 5회”, “황금똥 1회 회피” |
| 경고 시스템 | 새똥/보스 투사체/폭우 전 경고선 표시 |
| 보호 장치 | 우산 보호, 이어하기, 부활 무적 |
| 보상 구간 | 폭우 후 코인 보너스, 보스 격파 후 코인 샤워 |

## 4. Survivor.io 분석

### 4.1 핵심 한줄 정의

| 항목 | 내용 |
|---|---|
| 정의 | 한 손 이동만으로 적 무리를 피하며 자동 공격·레벨업·스킬 조합·장비 성장으로 반복하는 모바일 로그라이트 생존 액션 |
| 성공 공식 | 조작은 이동만, 재미는 성장 선택과 압박 상승에서 발생 |
| 제작 관점 | 세션 내 성장과 세션 밖 메타 성장이 분리되어 있다 |

### 4.2 30초 코어 루프

| 시간 | 플레이어 경험 | 설계 의미 |
|---:|---|---|
| 0~5초 | 이동만으로 적 접근 회피 | 입력 부담 최소화 |
| 5~15초 | 경험치/드롭 수집 | 성장 자원 확보 |
| 15~25초 | 레벨업/스킬 선택 | 세션 내 즉시 성장 |
| 25~30초 | 더 많은 적과 위험 대응 | 강해짐과 압박을 동시에 체감 |

### 4.3 시스템 분석

| 항목 | Survivor.io 방식 | 제작 템플릿 포인트 |
|---|---|---|
| 조작 | 한 손 이동 | 모바일 접근성 우수 |
| 공격 | 자동 공격 | DGP에서는 자동 공격 대신 파워업 자동 효과로 변환 |
| 난이도 | 적 수량, 웨이브, 보스, 시간 압박 | 시간 기반 밀도/속도/패턴 증가 |
| 성장 | 인게임 스킬 선택 + 장비 메타 | MVP는 가벼운 3택 보상까지만 |
| 세션 | 비교적 긴 생존형 | DGP는 1~3분 우선, 5분 이상은 엔드리스 |
| 아트 | 많은 객체를 밝고 단순하게 구분 | 화면 혼잡도 관리가 핵심 |

### 4.4 Don't Get Pooped! 적용

| 차용 요소 | 적용안 |
|---|---|
| 세션 내 성장 선택 | 30~45초마다 파워업 3택 팝업 |
| 보스 피크 | 45~60초 미니 보스, 2~3분 강한 보스 |
| 메타 재화 | 코인으로 스킨/배경/가벼운 능력 해금 |
| 장비 시스템 | 초기에는 제외, 시즌/장기 업데이트로만 고려 |

## 5. Super Hexagon 분석

### 5.1 핵심 한줄 정의

| 항목 | 내용 |
|---|---|
| 정의 | 좌/우 회전만으로 수렴하는 벽 패턴을 피하며 60초 생존을 목표로 하는 미니멀 고난이도 음악-반응형 아케이드 |
| 성공 공식 | 극단적으로 단순한 조작 + 읽을 수 있는 패턴 + 리듬 + 즉시 재도전 |
| 제작 관점 | 많은 콘텐츠보다 패턴 품질과 재시작 속도가 중요 |

### 5.2 30초 코어 루프

| 시간 | 플레이어 경험 | 설계 의미 |
|---:|---|---|
| 0~3초 | 벽이 접근하며 즉시 규칙 이해 | 설명보다 몸으로 학습 |
| 3~10초 | 좌/우로 빈틈 찾기 | 입력은 단순, 판단은 빠름 |
| 10~20초 | 패턴 반복/변형 인식 | 학습 가능한 블록 필요 |
| 20~30초 | 속도·색·음악 압박 증가 | 리듬과 긴장 강화 |
| 실패 후 | 즉시 재시도 | 실패 비용 최소화 |

### 5.3 시스템 분석

| 항목 | Super Hexagon 방식 | 제작 템플릿 포인트 |
|---|---|---|
| 조작 | 좌/우 2버튼 | 입력 수 제한의 극단적 사례 |
| 목표 | 60초 생존 | 명확한 마스터리 기준 |
| 난이도 | 속도, 회전, 패턴 복잡도 | 랜덤보다 패턴 세트가 중요 |
| 보상 | 최고 기록, 난이도 해금 | 실력 성장 중심 |
| 아트 | 기하학, 고대비, 미니멀 | 리소스 적어도 정체성 가능 |
| 오디오 | 강한 chiptune 리듬 | 음악은 텐션 장치 |

### 5.4 Don't Get Pooped! 적용

| 차용 요소 | 적용안 |
|---|---|
| 60초 목표 | “60초 생존” 도전 배지/업적 추가 |
| 패턴 블록 | 낙하 패턴 5~10개를 직접 설계 |
| 즉시 재시작 | 결과창에서 Retry를 가장 크게 배치 |
| 음악 싱크 | WARNING/보스/콤보 타이밍을 BPM과 맞춤 |

## 6. 게임 제작 템플릿에 반영할 최종 체크리스트

| 영역 | 체크리스트 |
|---|---|
| Concept | 한줄 정의가 `누가 / 무엇을 / 어떻게 / 왜 반복하는가`를 담는가 |
| Core Loop | 30초 안에 Action → Feedback → Reward → Retry가 도는가 |
| Input | 조작이 1개 또는 2개 이하인가 |
| Camera | 다음 위험을 읽을 수 있는 시야가 확보되는가 |
| Fairness | 경고 없는 즉사, 랜덤 안전지대 없음, 과도한 히트박스가 없는가 |
| Difficulty | 속도, 밀도, 패턴, 공간 압박을 분리해 튜닝 가능한가 |
| Reward | 점수, 코인, 미션, 수집을 분리했는가 |
| Meta | 성능 강화보다 코스메틱/기록/업적부터인가 |
| Art | 위험물/보상/플레이어가 즉시 구분되는가 |
| Audio | 수집/경고/피격/보스/게임오버 SFX가 분리되어 있는가 |
| QA | 5초/15초/30초/60초/120초 도달률을 확인하는가 |
| Release | 재시작 마찰, 로딩, 모바일 안전영역, 성능을 배포 전 확인하는가 |

## 7. Don't Get Pooped! 우선 반영 제안

| 우선순위 | 기능 | 근거 게임 | 이유 |
|---:|---|---|---|
| 1 | 낙하 예고 마커/그림자 | Crossy Road, Super Hexagon, Jetpack Joyride | 고난이도에서 공정성 확보 |
| 2 | 미션 3개 | Jetpack Joyride, Survivor.io | 최고점 외의 재도전 목적 부여 |
| 3 | 패턴 블록 스폰 | Super Hexagon, Crossy Road | 랜덤 억까 감소, 숙련감 증가 |
| 4 | 60초 챌린지/배지 | Super Hexagon | 짧고 명확한 목표 제공 |
| 5 | 파워업 3택 이벤트 | Survivor.io | 세션 내 성장감 추가 |
| 6 | 보스 연출 강화 | Jetpack Joyride, Survivor.io | 세션 피크와 보상 감정 강화 |
| 7 | 즉시 재시작 UX 개선 | 전체 | 반복 플레이 핵심 |
