# Game Archetype Recipes — 4 Reference Games → Reusable Production Tracks

> 목적: 레퍼런스 게임을 그대로 복제하지 않고, 그 구조를 재사용 가능한 “제작 레시피”로 변환한다. 각 레시피는 `게임 선정 → 시나리오 → 에셋 → 구현 → 테스트 → 배포` 순서를 따른다.

## 1. 공통 제작 레시피

```text
1) Reference DNA 추출
2) 내 게임의 한줄 피치 작성
3) 30초 코어 루프 정의
4) MVP 제외 범위 고정
5) 시나리오/스테이지/위험/보상 설계
6) 에셋 매니페스트 작성
7) 프롬프트/오디오/애니메이션 생성
8) Foundation 구현
9) Gameplay 구현
10) UX/메타/사운드 통합
11) QA/밸런싱
12) 배포/운영
```

## 2. Archetype A — Crossy Road형: 짧은 회피 + 수집 메타

### 2.1 게임 선정

| 항목 | 결정 |
|---|---|
| 적합한 장르 | 세로형 회피, 라인/타일 이동, 캐릭터 수집 |
| 핵심 재미 | 위험을 읽고 안전 경로를 선택하는 순간 판단 |
| MVP 한줄 | “한 손으로 위험 구간을 넘으며 기록과 캐릭터를 모으는 무한 아케이드” |

### 2.2 시나리오/시스템

| 설계 항목 | 내용 |
|---|---|
| 코어 루프 | 전진/이동 → 위험 관찰 → 안전 타이밍 선택 → 점수/코인 → 재시작 |
| 위험 | 차량, 강, 기차처럼 서로 다른 판정/속도의 장애물 |
| 난이도 | 차선 속도, 안전 공간, 정지 압박, 패턴 혼합 |
| 보상 | 거리 점수, 코인, 캐릭터/배경 수집 |

### 2.3 에셋

| 필수 에셋 | 수량 |
|---|---:|
| 플레이어 캐릭터 | 1 MVP / 8 확장 |
| 장애물 | 3 MVP / 10 확장 |
| 바닥/타일/배경 | 2 MVP / 8 확장 |
| 코인/수집물 | 1~3 |
| UI 버튼/결과창 | 6~10 |
| SFX | 이동, 코인, 충돌, 신기록, 버튼 |

### 2.4 구현 순서

1. 타일/라인 기반 월드 스크롤
2. 플레이어 입력과 이동 제한
3. 장애물 패턴 스폰
4. 충돌/점수/게임오버
5. 코인/캐릭터 해금
6. 즉시 재시작 UX
7. 일일 도전/이벤트

### 2.5 테스트

| 테스트 | 기준 |
|---|---|
| 공정성 | 모든 스폰 조합에 최소 1개 안전 경로 존재 |
| 시야 | 위험물이 진입 전 충분히 보임 |
| 반복성 | 사망 후 1탭 재시작 |
| 수집 | 코인 획득과 해금 저장이 유지됨 |

## 3. Archetype B — Jetpack Joyride형: 원터치 러너 + 미션/파워업

### 3.1 게임 선정

| 항목 | 결정 |
|---|---|
| 적합한 장르 | 자동 진행 러너, 원터치 고도/위치 조작 |
| 핵심 재미 | 간단한 입력으로 장애물과 수집물을 동시에 판단 |
| MVP 한줄 | “원터치로 위험을 피하고 코인·미션을 달성하는 빠른 러너” |

### 3.2 시나리오/시스템

| 설계 항목 | 내용 |
|---|---|
| 코어 루프 | 입력 → 위치 조절 → 장애물 회피 → 코인/파워업 → 미션 정산 |
| 위험 | 레이저, 미사일, 전기장처럼 경고 가능한 장애물 |
| 난이도 | 속도, 장애물 간격, 복합 패턴 |
| 보상 | 거리, 코인, 미션, 파워업, 런 종료 보너스 |

### 3.3 에셋

| 필수 에셋 | 수량 |
|---|---:|
| 주인공 | 1~3 |
| 장애물 | 3~5 |
| 파워업/차량/보호막 | 3 MVP / 8 확장 |
| 코인/토큰 | 2~3 |
| 연구소/배경 레이어 | 3~5 |
| SFX | 입력, 코인, 경고, 폭발, 파워업, 게임오버 |

### 3.4 구현 순서

1. 자동 스크롤/플레이어 물리
2. 원터치 입력
3. 장애물 스폰/경고
4. 코인 라인/파워업 박스
5. 미션 3개 시스템
6. 런 종료 정산
7. 상점/스킨/보상형 광고

### 3.5 테스트

| 테스트 | 기준 |
|---|---|
| 입력감 | 누름/뗌 반응이 지연 없이 동작 |
| 경고 | 유도/고속 위험은 0.4초 이상 예고 |
| 보상 밀도 | 30초 안에 코인/파워업/미션 중 2개 이상 경험 |
| 결과창 | 미션 진척, 코인, 최고점이 한눈에 보임 |

## 4. Archetype C — Survivor.io형: 한 손 생존 + 세션 내 성장

### 4.1 게임 선정

| 항목 | 결정 |
|---|---|
| 적합한 장르 | 자동 공격/자동 효과 기반 생존 로그라이트 |
| 핵심 재미 | 이동만 하면서 성장 선택으로 전투 양상이 바뀜 |
| MVP 한줄 | “한 손 이동만으로 몰려오는 위험을 피하고 매판 다른 빌드를 만드는 생존 게임” |

### 4.2 시나리오/시스템

| 설계 항목 | 내용 |
|---|---|
| 코어 루프 | 이동 → 위험 회피/처리 → 경험치/자원 수집 → 선택지 강화 → 웨이브 상승 |
| 위험 | 적 무리, 엘리트, 보스, 장판, 투사체 |
| 난이도 | 수량, 속도, 체력, 패턴, 보스 피크 |
| 보상 | 레벨업 3택, 장비/캐릭터/재화, 챕터 클리어 |

### 4.3 에셋

| 필수 에셋 | 수량 |
|---|---:|
| 플레이어 | 1~4 |
| 적 | 3 MVP / 15 확장 |
| 스킬 이펙트 | 3 MVP / 20 확장 |
| 경험치/상자/코인 | 3~5 |
| 보스 | 1 MVP / 5 확장 |
| UI | 선택 카드, 체력, 레벨, 결과, 장비 |
| SFX/BGM | 공격, 레벨업, 상자, 보스, 대량 처치, 승리/패배 |

### 4.4 구현 순서

1. 플레이어 이동과 카메라
2. 적 스폰/군집/충돌
3. 자동 공격 또는 자동 효과
4. 경험치/레벨업 선택
5. 웨이브/보스
6. 장비/캐릭터 메타
7. 이벤트/일일 도전

### 4.5 테스트

| 테스트 | 기준 |
|---|---|
| 화면 혼잡도 | 적/보상/위험이 겹쳐도 구분 가능 |
| 성장 체감 | 첫 30초 안에 강화 전후 차이가 보임 |
| 성능 | 많은 객체에도 FPS 안정 |
| 밸런스 | 빌드 선택이 의미 있지만 필수 정답 하나로 고정되지 않음 |

## 5. Archetype D — Super Hexagon형: 미니멀 패턴 + 고난이도 리듬

### 5.1 게임 선정

| 항목 | 결정 |
|---|---|
| 적합한 장르 | 1~2버튼 반응형 패턴 회피 |
| 핵심 재미 | 죽고 다시 하면서 패턴을 몸으로 학습 |
| MVP 한줄 | “좌우 조작만으로 60초 생존을 노리는 초고속 패턴 아케이드” |

### 5.2 시나리오/시스템

| 설계 항목 | 내용 |
|---|---|
| 코어 루프 | 시작 → 패턴 접근 → 좌/우 회피 → 기록 갱신 → 즉시 재시작 |
| 위험 | 회전벽, 수렴벽, 리듬 패턴, 색상/속도 변화 |
| 난이도 | 속도, 패턴 길이, 회전, 시야 압박 |
| 보상 | 최고 기록, 60초 클리어, 난이도 해금 |

### 5.3 에셋

| 필수 에셋 | 수량 |
|---|---:|
| 플레이어 도형 | 1~3 |
| 장애물 도형 | 5~10 패턴 블록 |
| 배경/색상 팔레트 | 난이도별 3~6 |
| 메시지 | Begin, Game Over, New Best, 60s Clear |
| 오디오 | 고BPM 루프, 실패음, 기록음, 패턴 진입음 |

### 5.4 구현 순서

1. 중심/회전 좌표계
2. 좌/우 입력
3. 패턴 블록 스폰
4. 충돌/생존 타이머
5. 난이도/속도 램프
6. 즉시 재시작
7. 음악 BPM 싱크

### 5.5 테스트

| 테스트 | 기준 |
|---|---|
| 재시작 | 실패 후 1초 이내 재시작 |
| 패턴 학습 | 같은 패턴을 반복하면 생존 시간이 늘어남 |
| 피로도 | 화면 회전/플래시가 과하지 않음 |
| 기록 | 0.1초 단위 기록 갱신이 명확히 보임 |

## 6. Don't Get Pooped!에 가장 맞는 혼합 레시피

`Don't Get Pooped!`는 네 레퍼런스를 모두 조금씩 섞되, 우선순위는 아래가 적합하다.

| 출처 | 가져올 것 | 비중 |
|---|---|---:|
| Jetpack Joyride | 미션, 파워업, 코인 정산, 짧은 세션 | 35% |
| Crossy Road | 수집 메타, 공정한 위험, 즉시 재시작 | 25% |
| Super Hexagon | 패턴 블록, 리듬감, 60초 도전 | 25% |
| Survivor.io | 세션 내 성장 3택, 보스 피크 | 15% |

### DGP용 다음 구현 순서

1. 공통 로딩/부트/오디오/세이브 Foundation 고정
2. 낙하 예고 마커/그림자
3. 패턴 블록 스폰 테이블
4. 미션 3개 시스템
5. 60초 챌린지/업적
6. 파워업 3택 이벤트
7. 보스 연출/HP/격파 보상 강화
8. 런 결과 분석 화면
