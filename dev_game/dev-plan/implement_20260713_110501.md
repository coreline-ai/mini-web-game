# implement_20260713_110501.md

작성 일시: `2026-07-13 11:05:01 KST`

이 문서는 10개 generated 게임의 후속 에셋 품질, 배포 용량, production asset 구조를 프로젝트별 병렬 에이전트가 안전하게 구현할 수 있도록 범위와 소유권, 선행 계약, 검증 순서를 고정하는 작업 문서다.

## 개발 목적

- `bullseye-rush`, `jungle-arcshot`의 현재 HQ screen 품질 게이트 실패를 기준 완화 없이 해소한다.
- 저장소 보존용 `_source`, `references`, `imagegen/raw`, `imagegen/sheets`가 Vite `dist`에 복사되는 문제를 manifest 기반 runtime allowlist로 해결한다.
- 10개 게임의 runtime 에셋과 source/provenance 에셋을 명시적으로 구분한다.
- 각 게임 에이전트가 자기 프로젝트만 수정하고 독립적으로 검증·커밋할 수 있는 병렬 작업 구조를 만든다.
- 최종 통합 시 10개 게임의 `1080x1920`, `scaleMode: fit`, imagegen provenance, gameplay 동작을 유지한다.

## 문서 기반 제약 사항

- 모든 게임은 `SPEC.canvas=1080x1920`, `scaleMode=fit`를 유지한다.
- `market-panic`, `meteor-dash`, `sky-archer`, `target-shooter-rush`는 accidental `3240x5760` DPR backing store가 다시 생기면 실패다.
- runtime 에셋은 각 게임의 `assets/**` 안에 있어야 하며 symlink와 다른 게임 참조를 금지한다.
- manifest의 `method: codex-gpt-imagegen-skill`, `sourceSkill: imagegen`, `promptHash`를 유지한다.
- `rawPath`, `sourceSheet`, `alphaSourceSheet`, `referenceAssets`는 provenance이며 runtime delivery 대상이 아니다.
- manifest가 runtime truth이고 loader, Vite delivery, 실제 network request가 서로 일치해야 한다.
- 단순 sharpen/noise 추가로 edge 수치만 높이는 보정은 허용하지 않는다.

## 결정 기록

### Runtime asset delivery 계약

- `asset-manifest.json`의 `stageBackgrounds`, `images`, `audio`, 필요 시 `hqScreenAssets` entry에 `delivery: "runtime" | "source"`를 명시한다.
- 암묵적 기본값을 두지 않는다. 누락된 delivery는 QA 실패로 처리한다.
- `delivery: runtime`의 `path`만 dev server와 build output에 제공한다.
- 동일 파일을 여러 entry가 선언하면 normalized path로 한 번만 배포하되 metadata 충돌은 실패시킨다.
- query-string cache buster는 비교에서 제거하지만 manifest 파일 경로에는 저장하지 않는다.
- `publicDir: false`와 package-local `scripts/runtime-asset-delivery.mjs`를 사용해 generated 게임의 self-contained 성격을 유지한다.
- package-local helper의 canonical template과 smoke test는 `generator/src/cli.mjs`가 소유한다.
- `dist/runtime-asset-manifest.json`에는 runtime `id/type/url/bytes/sha256`만 기록한다.

### Production asset tree 계약

- 권장 runtime image root는 `assets/images/production/**`다.
- manifest에는 `assets/...` 경로를, Phaser loader에는 기존 public root 규칙에 맞춰 `assets/`를 제거한 URL을 사용한다.
- `assetLayout.version = "production-images-v1"` marker가 있는 게임부터 strict path QA를 적용한다.
- `asset-plan.path`가 생성 원본이고 runtime이 후처리 파일이면 원본 path를 덮어쓰지 않고 `runtimePath`를 별도로 기록한다.
- 구조 이동 PR과 image regeneration PR은 분리한다.

### 선택형 신규 화면 에셋 결정

- 기본 결정은 `생성하지 않음`이다.
- 기존 캡처에서 화면 정체성이 부족하거나 scene-first artboard가 필수 계약이 되거나 composite/HQ gate가 실제로 실패할 때만 별도 imagegen PR로 승격한다.
- `meteor-dash`의 1170x2532 배경은 현재 FHD 이상이므로 resample artifact가 확인되지 않으면 유지한다.
