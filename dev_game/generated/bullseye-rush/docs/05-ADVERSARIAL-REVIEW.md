# 05 · Adversarial Review — Bullseye Rush

## "리스킨 아니냐"에 대한 반박 (구체 근거)
1. **스포너가 없다**: Foundation의 Spawner를 임포트조차 하지 않는다. 낙하물/스폰 흐름 대신 RoundManager의 라운드제 과녁 패턴(정지→이동→바운스→축소)이 콘텐츠를 구동한다.
2. **입력 문법이 처음 등장**: 드래그 이동(Sky)·드래그 조준(Castle)과 달리 조준이 완전 자동(왕복 스윕)이고 플레이어 입력은 탭 타이밍 하나다. 본 레포 4개 게임 중 유일한 타이밍 게임.
3. **실패 자원이 다르다**: 시간 생존(dodge)·거점 HP(Castle)가 아니라 **정확도 예산**(미스 3, 라운드별 화살 수). 생존 점수를 0으로 만들고 명중/콤보/클리어 보너스만으로 점수를 구성했다.
4. **승리 조건이 있다**: 무한 생존이 아니라 R12 올클리어 = WIN. 엔드 상태가 존재하는 첫 게임.
5. **신규 시스템 2개(OscillatingAim/RoundManager) 340여 줄**이 판정·경제·난이도 전부를 소유하고 런타임에 배선되어 있다. 에셋도 전부 이 게임 전용 신규 생성(과녁/별/화살/여궁수/사격장 배경 3종).

## 남는 리스크 (정직)
- Foundation shell(씬 구조/HUD/저장/오디오/스테이지 크로스페이드)은 재사용 — 이는 팩토리의 설계 의도.
- GameOver 씬이 승/패를 같은 화면으로 정산(WIN 연출은 플래시+사운드 차이뿐) — 확장 시 결과 화면 분기 권장.
