# implement_20260713_222027.md

작성 일시: `2026-07-13 22:20:27 KST`

이 문서는 `철갑특송: 라스트 라인` 실제 플레이 QA에서 확인된 완주 차단, 모바일 입력, 진행 안내, HUD·카메라 결함의 수정 범위를 고정하고 구현 순서를 명확히 하기 위한 작업 문서다.

## 개발 목적

현재 빌드를 “자동 Gate는 통과하지만 사람이 정상 입력으로 완주할 수 없는 상태”에서 다음 조건을 만족하는 실제 플레이 가능 상태로 수정한다.

- 디버그 teleport/advance 없이 처음부터 Result까지 정상 완주 가능
- 모바일에서 이동·사격·점프 조합을 동시에 사용할 수 있음
- 전투·호위 게이트가 진행 이유와 다음 행동을 명확히 표시함
- Pause의 모든 버튼이 보이고 정상 조작 가능
- HUD가 지원 viewport와 최악 문자열 상태에서 겹치지 않음
- 카메라가 진행 방향 전방 시야를 확보함
- 이번에 발견된 결함을 자동 회귀 검사가 다시 탐지함

## 개발 범위

- 지면과 고가 플랫폼의 충돌 책임 분리 및 전 구간 도달 가능성 수정
- 적이 플랫폼 측면에 고착되어 encounter를 영구 잠그는 문제 수정
- 전투 게이트 clamp 진동 제거와 잔존 적 방향 안내
- 호위 차량 이동 조건 수정과 진행률·거리·복귀 방향 안내
- 모바일 멀티터치 활성화와 포인터별 입력 소유권 정리
- Pause 버튼 depth 및 재개·철수 입력 복구
- HUD 데스크톱/모바일 반응형 재배치와 동적 문자열 폭 처리
- 카메라 look-ahead 방향 및 전·후진 화면 위치 수정
- 물리 좌표·중력 설정 단일화
- 실제 입력 기반 완주, 멀티터치, HUD, Pause, 게이트, 플랫폼 회귀 검사 추가
- QA 요약과 회귀 체크리스트 갱신
