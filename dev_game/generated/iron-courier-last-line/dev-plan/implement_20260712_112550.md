## 개발 목적

현재의 기능형 프로토타입을 모바일 우선 production demo 품질로 보강한다. 사용자가 지적한 다음 문제를 실제 고화질 이미지 에셋과 런타임 애니메이션으로 해결한다.

- 코드 생성 지형·구조물·구조 대상이 최종 화면에 노출되는 문제
- 하단의 반복 점선 지형이 이동 경로처럼 보이는 문제
- 16,800px 월드에서 배경이 연속 이동하지 않고 두 경계에서 갑자기 교체되는 문제
- 적·플레이어·차량·보스가 정지 이미지 또는 변형만으로 움직이는 문제
- projectile, pickup, VFX, HUD, 모바일 컨트롤이 placeholder/기본 도형에 의존하는 문제
- manifest에 선언된 일부 파일만 검사하여 전체 에셋 누락을 놓치는 QA coverage 문제

완료 시점에는 gameplay layer의 미승인 placeholder가 0건이고, 배경·지형·캐릭터·구조 대상·전투 효과·차량·보스·UI가 하나의 아트 디렉션과 명시적 상태 계약을 사용해야 한다.

## 문서 기반 제약 사항

- 모바일 가로 화면을 1순위로 하고 PC 키보드를 2순위 호환으로 유지한다.
- 모든 신규 이미지는 이 게임 전용 에셋이어야 하며 공유 에셋·symlink를 사용하지 않는다.
- 이미지 생성은 Codex 내장 `imagegen` 경로를 사용하고 raw/cleaned/runtime 결과와 provenance를 보존한다.
- 애니메이션 셀은 같은 크기, 동일 baseline/root anchor, 동일 정체성, 투명 배경을 지킨다.
- 텍스트와 숫자는 이미지에 굽지 않고 runtime text로 유지한다.
- 배경은 gameplay entity를 포함하지 않으며 far/mid/near 소유권을 분리한다.
- 시각 문제를 코드 tint, 회전, scale 변형으로 우회하지 않고 필요한 source asset/state를 제작한다.
- defect 수정 전 기준 캡처를 보존하고 수정 후 같은 위치·viewport·입력 조건으로 다시 캡처한다.

## 공통 진행 규칙

- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 시작한다.
- 구현 중 발생한 이슈는 해당 Phase에서 수정하고 기록한다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
- 요구사항이 불명확하면 가능한 케이스를 나누고 확인 후 진행한다.
- 한 기능은 유지보수 가능한 최소 책임 단위로 나누고, 각 단위의 역할과 경계를 계획에 적는다.
- 계획 밖 옵션·확장·부가 기능을 만들기 위한 분리는 하지 않는다.
- 공통화는 계획 범위 안의 중복을 줄일 때만 수행하고 과도한 추상화는 피한다.
- 기존 Phaser API와 프로젝트 시스템을 우선 사용하고 새 runtime 의존성은 추가하지 않는다.
- gameplay visual에서 `__WHITE`, 임시 Graphics texture, 미승인 primitive 사용을 금지한다.
- 코드 네이티브 text와 동적 HP fill은 허용하되 frame/panel/icon의 이미지 소유권을 명확히 기록한다.
- 한 에셋 family가 승인되기 전 대량 변형 생성을 시작하지 않는다.
- 수정 후 원래 재현 조건의 before/after 증거와 상태 JSON이 없으면 완료 처리하지 않는다.

## 에셋 제작 공통 계약

### 캐릭터·애니메이션

- source frame: 1024×1024 이상 권장, runtime virtual cell 512×512 또는 역할별 고정 셀
- 방향: 기본 우측 완전 측면, 좌측은 runtime flip 사용
- 공통 baseline과 발 사이 root anchor 유지
- body/weapon 8% 이상 safe padding, 인접 셀 불투명 픽셀 비접촉
- 총구 화염·탄피·그림자·연기는 본체 프레임에 굽지 않는다.
- animation frame event와 projectile/hit 판정을 동기화한다.

### 배경·지형

- far/mid/near를 별도 에셋으로 만들고 플레이어·적·차량·충돌 구조를 제외한다.
- 구역 경계는 960~1,280px spatial blend를 사용한다.
- 16,800px 전 구간의 horizon, 지면 높이, 광원 방향을 연속 유지한다.
- 지형 이미지는 collision 상단선, cap, center tile, support를 분리하며 비정상 stretch를 금지한다.

### UI·VFX

- UI frame/panel은 9-slice 또는 정비율 이미지를 사용하고 margin metadata를 기록한다.
- 텍스트 없는 button base/icon을 제작하며 pressed/disabled 상태를 포함한다.
- VFX는 판정 가독성을 가리지 않고 소·중·대 또는 피해 재질별 변형을 제공한다.

### 저장·Manifest

- 원본: `assets/source/raw/`
- 정리본: `assets/source/cleaned/`
- animation frame: `assets/source/animation-frames/`
- runtime: `assets/runtime/`
- atlas/metadata: `assets/runtime/atlases/`
- QA: `assets/qa/`
- 모든 최종 에셋은 `asset-manifest.json`에 family/state/frame/pivot/sourceSize/runtimeSize/status/provenance를 기록한다.

## 잔여 리스크 / 후속 과제

- AI 생성 다중 프레임의 identity drift와 손·무기 변형 위험이 가장 크므로 reference 승인과 소량 batch QA가 필요하다.

- parallax 배경에 구워진 구조와 runtime terrain의 Visual Singularity 중복을 지속 검사해야 한다.
- 보스 파츠 기반 전환은 pivot·attachment 오차와 내부 alpha hole을 별도 검사해야 한다.
- 사람 기준 7~10분 median clear time과 실제 iOS/Android 발열·오디오 unlock은 이 계획 완료 후 배포 후보 단계에서 측정한다.
