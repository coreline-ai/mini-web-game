# Skybreak Gunship — 실행 화면 QA 조사 (2026-07-14)

## 대상

- URL: `http://127.0.0.1:5187/?training=1`
- 뷰포트: 390×844 모바일, 1280×720 데스크톱 임베드
- 범위: 홈 → 브리핑 → 3단계 실전 훈련 진입, 조준 입력, 전투 회귀

## 발견 사항

### P2 — 조준 입력과 안내 문구의 기준점 불일치

`AimSystem`이 `aimOffsetY: 42`를 적용해 손가락 좌표보다 조준점이 42 논리 픽셀 위에 표시된다. 따라서 사용자가 안내 문구대로 적 스프라이트 중앙을 직접 드래그하면 조준점은 적 위에 멈추고 `실전 훈련 1/3`이 진행되지 않는다. 적보다 약 42px 아래를 드래그해야 조준점이 적 중앙에 맞고 2/3으로 진행된다.

- 재현 캡처: `skybreak-gunship/qa-captures/manual-iab-direct-target-offset.png`
- 정상 진행 캡처: `skybreak-gunship/qa-captures/manual-iab-aligned-aim.png`
- 사용자 영향: 첫 조작에서 “입력이 먹지 않는다/멈췄다”로 인식할 수 있음
- 권장 조치: 튜토리얼에서 오프셋을 시각적으로 표시하거나, 훈련 단계의 판정만 `pointer` 기준으로 보정해 문구와 실제 조작 기준을 일치시킨다.

### P2 — 1단계 안내 화살표가 건십과 겹쳐 식별성이 낮음

1단계 화살표가 논리 좌표 y=211에 있고 건십 스프라이트(중심 y=214, 높이 100)와 같은 영역을 차지한다. 깊이 레이어상 표시되지만 건십의 금속 디테일과 겹쳐 짧은 프레임에서는 화살표가 거의 보이지 않는다. 2단계 이후 버튼 화살표보다 안내성이 약하다.

- 재현 캡처: `skybreak-gunship/qa-captures/manual-iab-mobile-combat.png`
- 권장 조치: 1단계 화살표를 적 마커 방향(또는 패널 하단의 별도 여백)으로 이동하고, 건십과 최소 24px의 시각적 간격을 확보한다.

### 환경 관찰 — 데스크톱 임베드의 좌우 레터박스

1280×720 임베드에서 캔버스는 390:844 FIT 비율을 유지해 CSS 약 332.5×720으로 중앙 배치된다. 좌우 검은 여백이 크게 보이지만, 모바일 세로 게임의 의도된 `FIT` 동작이며 390×844에서 콘텐츠는 정상적으로 꽉 찬다. 데스크톱에서도 가로로 꽉 찬 화면을 요구한다면 별도 `cover`/반응형 프레젠테이션 정책이 필요하다.

### 환경 관찰 — `training=1`은 튜토리얼을 매번 강제

현재 URL의 `training=1` 쿼리가 `SaveData` 완료 여부와 무관하게 실전 훈련을 다시 연다. 따라서 미션 타이머가 0초에서 멈춘 것처럼 보이는 것은 해당 QA URL의 의도된 동작이다. 일반 플레이 확인 시 쿼리를 제거해야 한다.

## 결론

전투 시스템·브라우저 오류·메모리/입력 회귀는 PASS다. 현재 화면에서 실제로 이상하게 느껴질 핵심은 **조준점 오프셋이 안내에 노출되지 않는 점**과 **1단계 안내 화살표가 건십과 겹치는 점**이다. 데스크톱 좌우 여백과 `training=1`은 결함이 아니라 현재 실행 조건에 따른 표시다.
